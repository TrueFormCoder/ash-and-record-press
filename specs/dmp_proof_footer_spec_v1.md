# DMP Proof Footer Spec v1.0

Mandatory metadata block for every DMP episode.
See full spec: DMP_Proof_Footer_Spec_v1.md

## Required Fields Per Episode
1. Source Construct (which Mirror OS column drives this)
2. Relevant Fields (exact column names)
3. Diagnostic Trigger (what conditions produce this pattern)
4. Counter-Pattern (what looks similar but is different)
5. What This Does Not Claim (anti-overreach)
6. Likely Public Misread (meme-flattening protection)
7. Intervention Implication (what a practitioner would do)
8. Confidence Tier
9. Doctrine Version

## Why It Matters
- Anti-overreach prevents self-intoxication
- Misread protection prevents meme-flattening
- Voluntary limits = court-grade credibility
